When creating pages in the Atomic Learning PLatform, you may wish to include either in-line snippets of code or code blocks which span several lines.

In-line code snippets

To create an inline code snippet, wrap the code in the <code> tags. For example, this code snippet console.log("Hello, World!"); was created using the following HTML source code:

...this code snippet <code>console.log("Hello, World!");</code> was created...

Code blocks

To insert a longer multi-line block of code, wrap the code in both <pre> and <code> tags. For example, this code block:

function greet(name) {
    console.log("Hello, " + name + "!");}

was created using the following HTML source code:

...this code block:</p>

<pre><code>function greet(name) {
    console.log("Hello, " + name + "!");}</code></pre>

<p>was created...

Note that the <pre> tag preserves whitespace and line breaks, which preserves indentation.

Minimising Blank Lines

To minimise blanks lines which appear within a code block, ensure that the first line of the code in the block directly follows the <pre><code> tag and that the closing </code></pre> tag is on the same line as the last line of the code.

Runnable Code Cells

For some language, such as Python and R, the Atomic Learning Platform supports the inclusion of runnable code cells, which are detailed in pages in the Related Content section.

Syntax Highlighting

For inline code snippets or code blocks, syntax highlighting can be applied by giving the <code> tag a class attribute with the word language- followed by the identifier associated with the language. For example:

<code class="language-python">print("Hello, World!")</code>

will render as print("Hello, World!").
